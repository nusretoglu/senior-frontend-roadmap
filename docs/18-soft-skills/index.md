# Soft Skills - Senior Developer Yo'lida

## Kirish

Texnik bilim - bu faqat yarimsharoba. Haqiqiy senior developer bo'lish uchun **soft skills** (yumshoq ko'nikmalar) shart. Bu bo'limda siz kod yozishdan tashqari eng muhim professional ko'nikmalarni o'rganasiz.

## Nega Soft Skills Muhim?

```
Junior Developer:  Kod yozadi
Mid-Level:         Yaxshi kod yozadi
Senior Developer:  Jamoa bilan birgalikda yaxshi tizimlar quradi
```

**Statistika:**
- Google'ning Project Aristotle tadqiqoti: eng samarali jamoalarda texnik mahoratdan ko'ra **psixologik xavfsizlik** muhimroq
- Stack Overflow Survey: 70%+ ishga olish qarorida soft skills hal qiluvchi omil
- Senior pozitsiyalarda 40-60% vaqt **kommunikatsiya**ga sarflanadi

## Bo'lim Tarkibi

### 1. [Code Review](./01-code-review.md)
Kod ko'rib chiqish - bu faqat xatolarni topish emas. Bu bilim almashish, mentoring va jamoa standartlarini shakllantirish.

**O'rganasiz:**
- Samarali review berish va qabul qilish
- Konstruktiv tanqid san'ati
- Review tezligi va sifati balansi

### 2. [Mentoring](./02-mentoring.md)
Boshqalarni o'stirish - senior developerning eng muhim vazifalaridan biri.

**O'rganasiz:**
- Junior va mid-level dasturchilarni yo'naltirish
- Feedback berish texnikasi
- Delegatsiya va ishonch

### 3. [Task Breakdown](./03-task-breakdown.md)
Katta muammolarni kichik, boshqariladigan qismlarga bo'lish san'ati.

**O'rganasiz:**
- User story yozish
- Estimation texnikalari
- Scope creep bilan kurashish

### 4. [Communication](./04-communication.md)
Texnik va notexnik odamlar bilan samarali muloqot.

**O'rganasiz:**
- Texnik tushunchalarni oddiy tilda ifodalash
- Yozma va og'zaki kommunikatsiya
- Konflikt hal qilish

### 5. [Architecture Discussion](./05-architecture-discussion.md)
Arxitektura qarorlarini muhokama qilish va himoya qilish.

**O'rganasiz:**
- RFC va ADR yozish
- Trade-off tahlili
- Stakeholderlar bilan ishlash

### 6. [Interview Tips](./06-interview-tips.md)
Texnik intervyudan muvaffaqiyatli o'tish va o'zingiz intervyu olish.

**O'rganasiz:**
- System design intervyu
- Behavioral savollarga javob
- Intervyu olish texnikasi

## Senior Mindset: Asosiy Tamoyillar

### 1. Ownership (Egalik Hissi)
```
Junior:  "Bu mening vazifam emas"
Senior:  "Bu bizning loyihamiz, men yordam beraman"
```

### 2. Impact Thinking (Ta'sir Haqida O'ylash)
```
Junior:  "Qanday qilib bu feature'ni implement qilaman?"
Senior:  "Bu feature biznesga qanday ta'sir qiladi?"
```

### 3. Long-term Vision (Uzoq Muddatli Ko'rish)
```
Junior:  "Bugun ishlaydi - yetarli"
Senior:  "6 oydan keyin buni maintain qilish qanday bo'ladi?"
```

### 4. Empathy (Hamdardlik)
```
Junior:  "Nega bu oddiy narsani tushunmayapti?"
Senior:  "Men ham bir paytlar shu joyda edim"
```

### 5. Continuous Learning (Doimiy O'rganish)
```
Junior:  "Men Vue bilaman"
Senior:  "Men Vue bilaman, lekin React/Svelte/Solid ham o'rganmoqdaman"
```
